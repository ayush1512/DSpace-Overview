# Comprehensive DSpace Project Report

## 1. Project Overview
DSpace is an open-source digital asset management system used by thousands of institutions worldwide to provide durable access to digital research, data, and scholarly work. It is built with a Java-based backend providing a comprehensive REST API and an Angular-based frontend.

## 2. Architecture & Modules
The project is divided into several key Java modules:

### 2.1 Backend (Java)
*   **`dspace-api`**: The core business logic layer. It contains the domain model and service layer.
*   **`dspace-server-webapp`**: The REST API layer, handling HTTP requests and managing the Spring Web application context.
*   **`dspace-oai`**: OAI-PMH protocol support for metadata harvesting and exposing.
*   **`dspace-sword` / `dspace-swordv2`**: Support for the SWORD deposition protocol.
*   **`dspace-iiif`**: IIIF (International Image Interoperability Framework) support.
*   **`dspace-rdf`**: RDF serialization of DSpace objects.

### 2.2 Frontend (Angular)
*   Located in `DSpace/dspace-angular`. It serves as the user interface and communicates exclusively with the backend REST API.

## 3. Key Functional Components & Files

### 3.1 Content Management (`dspace-api/src/main/java/org/dspace/content`)
*   **Core Entities**: `DSpaceObject.java`, `Item.java`, `Collection.java`, `Community.java`, `Entity.java`.
*   **Metadata Handling**: `MetadataSchema.java`, `MetadataField.java`, `MetadataValue.java`, `DCDate.java`, `DCLanguage.java`.
*   **Bitstream & Bundle Management**: `Bitstream.java`, `Bundle.java`, `BitstreamFormat.java`, `BitstreamServiceImpl.java`.
*   **Relationships**: `Relationship.java`, `RelationshipType.java`, `RelationshipMetadataService.java`.
*   **Authority & Vocabulary**: `authority/`, `vocabulary/` subdirectories for controlled vocabulary and metadata authority management.

### 3.2 REST API Controllers (`dspace-server-webapp/src/main/java/org/dspace/app/rest`)
*   **Resource Controllers**: `RestResourceController.java`, `ItemRestController.java`, `CollectionRestController.java`, `CommunityRestController.java`.
*   **Discovery & Search**: `DiscoveryRestController.java` (integrates with Solr).
*   **Bitstream Operations**: `BitstreamRestController.java`, `BitstreamReplaceController.java`.
*   **Submission**: `ItemTemplateRestController.java`, `SubmissionCCLicenseUrlRepository.java`.
*   **Authentication & Authorization**: `AuthenticationRestController.java`, `GroupRestController.java`, `ResourcePolicyEPersonReplaceRestController.java`.
*   **Statistics**: `StatisticsRestController.java`.
*   **Workflow Management**: `WorkflowDefinitionCollectionsLinkRepository.java`, `WorkflowStepActionsLinkRepository.java`.

## 4. Key Features
1.  **Hierarchical Repository**: Supports a Community -> Collection -> Item hierarchy for organizing digital assets.
2.  **Rich Metadata**: Extensible metadata schema (Dublin Core + custom fields) and crosswalk capabilities.
3.  **Bitstream Management**: Supports versioning, categorization, and replacement of binary files (PDFs, images, datasets).
4.  **Relationships**: Links between items (e.g., citing/cited, author/thesis).
5.  **OAI-PMX & SWORD**: Protocols for external harvesting and deposit.
6.  **Access Control**: Role-based access control (RBAC) with specific policies for read, write, and admin permissions.
7.  **Search & Discovery**: Apache Solr integration for powerful, faceted search.
8.  **Statistics & Usage Tracking**: Tracks download and view counts.
9.  **Authentication**: Supports local login, LDAP, Shibboleth, SAML, and OIDC.

## 5. Configuration & Persistence
*   **Database**: PostgreSQL.
*   **ORM**: Hibernate (JPA) for database persistence.
*   **Framework**: Spring Framework (MVC, Security, Configuration).
*   **Configuration Files**: `dspace.cfg` and Spring context files in `config/`.

## 6. Deployment Options
*   **Traditional**: Deploy as a WAR file to a servlet container (e.g., Apache Tomcat or Jetty).
*   **Docker**: Provided `Dockerfile` and `docker-compose.yml` for development and testing.
*   **Kubernetes**: Supported via Docker/Kubernetes for production cloud environments.

## 7. Testing
*   **Unit Tests**: Run via Maven Surefire (`-DskipUnitTests=false`).
*   **Integration Tests**: Run via Maven Failsafe (`-DskipIntegrationTests=false`).
*   **Test Fixtures**: Database test fixtures (H2/HSQLDB) for mocking the persistence layer.
*   **CI/CD**: Automated builds and tests via GitHub Actions.

## 8. Summary
DSpace is a mature, highly modular Java-based repository system. It features a robust REST API, a flexible and XML-configurable submission workflow, and standard protocol support (OAI-PMH, SWORD, IIIF). Its clear separation of concerns across the API, converter, repository, and business logic layers makes it extensible and suitable for large-scale institutional repositories.
