# DSpace Project Architecture Overview

## 1. Backend Components (Java)

### `dspace-api` (Core Business Logic)
- Contains the core domain model, business logic, and service layer.
- **Packages (`org.dspace.*`)**:
  - `content`: Item, Collection, Community management, metadata handling, bitstream management.
  - `eperson` & `authenticate`: User management, authentication, and policy management.
  - `authorize`: Authorization and permission checking (ResourcePolicy, Group).
  - `workflow` & `xmlworkflow`: Submission workflow management (XML-defined or simple).
  - `submit`: Item submission process.
  - `discover`: Discovery and search functionality (integration with Solr).
  - `identifier`: Handle/UUID identifier management.
  - `storage`: Bitstream storage and retrieval.
  - `statistics`: Usage statistics and tracking.
  - `search`: Search index integration.
  - `oai`: OAI-PMX protocol support.
  - `sword` & `swordv2`: SWORD protocol support.
  - `iiif`: IIIF image protocol support.

### `dspace-server-webapp` (REST API Layer)
- Contains REST API controllers and web application configuration.
- **`org.dspace.app.rest`**: REST API layer.
- **`repository`**: Repository classes (data access layer for JPA/Hibernate).
- **`converter`**: Converts between API models and domain objects.
- **`link`**: HAL link building utilities.
- **`security`**: Spring Security configuration.
- **`filter`**: Request/response filters.
- **`login`**: Login session handling.

### Supporting Modules
- `dspace-services`: Configuration and service integration.
- `dspace-oai`: OAI-PMH protocol support.
- `dspace-sword` & `dspace-swordv2`: SWORD deposition protocols.
- `dspace-rdf`: RDF serialization of DSpace objects.
- `dspace-iiif`: IIIF image serving.
- `dspace-saml2`: SAML 2.0 authentication support.

## 2. Frontend (Angular)
- Located in `DSpace/dspace-angular` repository.
- Provides the user interface.
- Communicates with the backend via the REST API.

## 3. Key Functional Components & REST Controllers
- **`ItemRestController`**, **`CollectionRestController`**, **`CommunityRestController`**: CRUD operations for repository hierarchy.
- **`EPersonRestController`**: User management and profiles.
- **`BitstreamRestController`**: Bitstream (file) operations and replacement.
- **`RelationshipRestController`**: Item relationships (e.g., citing/cited).
- **`DiscoveryRestController`**: Discovery/search endpoint.
- **`StatisticsRestController`**: Usage statistics.
- **`AuthenticationRestController`**: Authentication endpoints (LDAP, Shibboleth, OIDC).
- **`GroupRestController`**: Group management.

## 4. Architecture Layers
- **API/Controller Layer**: Handles HTTP requests and maps them to services.
- **Converter Layer**: Converts between API JSON representations and internal Java objects.
- **Model Layer (`org.dspace.app.rest.model`)**: API model objects representing REST resources.
- **Repository Layer (`repository`)**: Data access objects (DAOs) interacting with the database via JPA/Hibernate.
- **Service/Business Layer**: Core business logic and domain objects.

## 5. Configuration & Database
- Uses Spring Framework and Hibernate (JPA) for persistence.
- **Database**: PostgreSQL.
- **Configuration**: `dspace/config/` directory (`dspace.cfg`, Spring context, Hibernate mapping).
- Manages connection pooling and ORM mapping.

## 6. Key Features & Capabilities
1. **Content Management**: Hierarchical structure (Community > Collection > Item) with rich metadata support.
2. **Access Control**: Role-based access control, resource policies, and bitstream-level security.
3. **Submission Workflow**: Configurable multi-step workflows with approval steps.
4. **Discovery & Search**: Full-text search via Apache Solr with faceted search capabilities.
5. **OAI-PMH**: Metadata harvesting and exposing protocol support.
6. **SWORD**: External deposit and batch upload support (v1 & v2).
7. **Statistics & Usage**: Download and view statistics tracking.
8. **Authentication**: Local, LDAP, Shibboleth, SAML, OIDC, and external authentication plugins.
9. **IIIF**: Standard image protocol support.

## 7. Deployment & Testing
- **Deployment**: WAR file deployment (Tomcat), Docker Compose (development), Docker/Kubernetes (production).
- **Testing**:
  - Unit tests (Maven Surefire).
  - Integration tests (Maven Failsafe).
  - Database test fixtures (Flyway/H2/HSQLDB for testing).
- **CI/CD**: GitHub Actions for automated builds, tests, and releases.

## 8. Summary
DSpace is a mature, modular Java-based digital repository system with a separate Angular frontend. It provides a robust REST API, supports multiple external protocols (OAI-PMH, SWORD, IIIF), and utilizes standard web technologies (Spring, Hibernate, PostgreSQL) for persistence and web serving. The architecture follows a clear separation of concerns across the API, converter, repository, and business logic layers.
